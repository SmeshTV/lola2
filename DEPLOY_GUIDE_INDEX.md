# 📚 Инструкции по Deploy - Обзор файлов

Все инструкции находятся в следующих файлах:

## 📄 Файлы инструкций

### 1. **QUICK_DEPLOY.md** ⚡ (НАЧНИ С ЭТОГО)
Быстрая шпаргалка на одну страницу. 
**Для тех кто в спешке!**

```bash
# Суть в 3 команды:
npm run build
git add . && git commit -m "description"
git push
```

---

### 2. **DEPLOYMENT.md** 📖 (ПОДРОБНО)
Полная инструкция по процессу deploy с объяснениями.
**Для первичной настройки и понимания процесса**

Содержит:
- ✅ Оба варианта обновления
- ✅ Полная настройка Netlify
- ✅ Чеклист перед deploy
- ✅ Что делать если что-то не работает

---

### 3. **NETLIFY_ENV_SETUP.md** 🔐 (ПЕРЕМЕННЫЕ)
Как добавить переменные окружения в Netlify.
**Для настройки Supabase и Discord ключей**

Содержит:
- ✅ Способы добавления переменных
- ✅ Откуда взять каждую переменную
- ✅ Таблица со всеми переменными
- ✅ Проверка что всё работает

---

### 4. **netlify.toml** ⚙️ (КОНФИГ)
Автоматическая конфигурация Netlify.
**Уже создан, используется автоматически**

Что делает:
- Команда сборки: `npm run build`
- Папка для публикации: `dist`
- Кэширование файлов
- SPA редирект для маршрутизации

---

### 5. **.env.example** 📋 (ШАБЛОН)
Шаблон переменных окружения.
**Используй как справку**

---

## 🚀 Быстрый старт (3 минуты)

### Первый раз настройка:

1. **Прочитай** → `QUICK_DEPLOY.md` (1 мин)
2. **Выполни** → команды для первого deploy (2 мин)
3. **Настрой** → переменные окружения в Netlify (используй `NETLIFY_ENV_SETUP.md`)

### Обычный процесс разработки:

1. Делай изменения в коде
2. Тестируй локально: `npm run dev`
3. Собирай: `npm run build`
4. Пушь: `git push`
5. Netlify автоматически задеплоит! ✅

## ❓ Что дальше?

### Если ты новичок:
1. Прочитай `QUICK_DEPLOY.md`
2. Выполни 3 команды
3. Проверь что задеплоилось на Netlify

### Если чего-то не работает:
1. Смотри раздел **"Если что-то не работает"** в `DEPLOYMENT.md`
2. Или проверь `NETLIFY_ENV_SETUP.md` если проблема с переменными

### Если нужна полная информация:
Прочитай `DEPLOYMENT.md` полностью
